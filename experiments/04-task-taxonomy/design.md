# 実験04: タスク分類別エージェント適性マトリクス

## 目的
コーディングタスクだけでなく、実務的タスク・コンテンツ制作・クリエイティブタスクまで拡張し、Claude Code と Codex の適性領域を包括的にマッピングする。

## 外部エビデンス（ベンチマーク参照）
- Terminal-Bench 2.0: Codex 75.1% > Claude 65.4% （CLI直線型はCodex優位）
- OSWorld-Verified: Claude 72.7% > Codex 64.7% （GUI/文脈理解はClaude優位）
- 出典: smartscope.blog (2026-02-15)

## タスク分類と実験設計

### カテゴリA: コーディングタスク
| 実験 | タスク | 予測 |
|------|--------|------|
| 4A-1 | バグ修正（テスト駆動） | Codex有利（CLI直線型） |
| 4A-2 | リファクタリング（既存コードスタイル準拠） | Claude有利（文脈統合型） |
| 4A-3 | 新機能実装（既存アーキテクチャ理解必要） | Claude有利（探索型） |

### カテゴリB: 実務的タスク
| 実験 | タスク | 予測 |
|------|--------|------|
| 4B-1 | READMEの自動生成（コードから） | Codex有利（構造解析） |
| 4B-2 | CHANGELOG作成（git log + コード変更の要約） | 互角 |
| 4B-3 | コードレビューコメント作成（ADR準拠チェック） | Claude有利（文脈+表現） |

### カテゴリC: コンテンツ制作タスク
| 実験 | タスク | 予測 |
|------|--------|------|
| 4C-1 | 技術ブログ記事のドラフト（実験結果をもとに） | Claude有利（表現力+文脈） |
| 4C-2 | X投稿スレッドの作成（フック+展開+CTA） | Claude有利（トーン制御） |
| 4C-3 | API ドキュメント生成（OpenAPI spec → 人間可読文書） | Codex有利（構造変換） |

### カテゴリD: クリエイティブタスク
| 実験 | タスク | 予測 |
|------|--------|------|
| 4D-1 | 人格付きチャットボットの応答生成 | Claude有利（人格持続） |
| 4D-2 | コードコメントにユーモアを入れる | Claude有利（トーン） |
| 4D-3 | エラーメッセージを親しみやすく書き直す | Claude有利（UXライティング） |

## 評価基準

### 定量
- 完了時間（秒）
- トークン使用量
- タスク達成率（基準を満たしたか）

### 定性
- 表現の自然さ（1-5）
- 文脈の反映度（1-5）
- 人格/トーンの一貫性（1-5）
- 独自の洞察・付加価値（1-5）

## 仮説マトリクス

```
              CLI直線型 ←――――→ 文脈統合型
              ↓                    ↓
      Codex最適化ゾーン    Claude最適化ゾーン
              |                    |
  [4A-1] バグ修正        [4A-2] リファクタ
  [4B-1] README生成      [4B-3] レビューコメント
  [4C-3] API文書生成      [4C-1] ブログ記事
                          [4C-2] X投稿
                          [4D-*] クリエイティブ全般
```

## 実行計画
- Phase 1: カテゴリA（コーディング）3実験 ← 実験02,03で部分完了
- Phase 2: カテゴリC（コンテンツ制作）3実験 ← 最もインサイトが出る領域
- Phase 3: カテゴリB（実務）+ カテゴリD（クリエイティブ）
