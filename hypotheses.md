# 仮説一覧

## H1: コンテキストチェーン仮説

**主張**: Claude Codeは指示ファイルから外部ファイルへの参照（「詳細は docs/overview.md を参照」）を自律的に追跡してファイルを読みに行くが、Codex CLIはAGENTS.mdに直接記載された情報のみを使用し、外部ファイルへの参照を追跡しない。

**根拠**:
- Claude Code公式: 子ディレクトリのCLAUDE.mdはオンデマンドでロードされる設計
- Codex公式: AGENTS.mdは起動時に1回だけ読み込み、32KB上限で切り捨て

**予測**:
- CLAUDE.mdに「docs/overview.md を見よ」と書けば、Claudeはoverview.mdを自発的に読む
- AGENTS.mdに同じ参照を書いても、Codexはoverview.mdを読みに行かない

**予備実験結果**: さっきの実験でCodexは `docs/overview.md` の存在を認識したが自発的に読みに行かなかった（AGENTS.mdにバグ情報をインラインで全て記載したため不要だった）。次の実験ではバグ情報を外部ファイルにのみ記載して検証する。

---

## H2: 暗黙的バグ発見仮説

**主張**: 指示ファイルにバグの存在を明示しない場合、Claude Codeはプロジェクト内のドキュメント（overview.md等）を自発的に発見・参照してバグ情報を取得するが、Codexはソースコード内のコメントのみに依存する。

**根拠**:
- Claude Codeはファイルシステムを探索的に読む傾向がある（ツールを自然にチェーンする）
- Codexはタスクを最短経路で解決する傾向がある（SWEベンチ最適化）

**予測**:
- Claude: overview.mdを発見→バグ情報取得→修正（2/2バグ発見）
- Codex: ソースコードのBUGコメントのみ発見→修正（2/2だが根拠が異なる）
- もしソースコードからBUGコメントも削除した場合、Claudeの方がバグ発見率が高い

---

## H3: 人格持続性仮説

**主張**: SOUL.mdやAGENTS.mdに記載された人格指示（口調、キャラクター設定）は、技術タスクの認知負荷が高いとき両者とも減衰するが、Claude Codeの方が持続性が高い。

**根拠**:
- Claude Codeは複数のコンテキストファイルを常時保持する設計
- Codexは32KB上限があり、技術的指示が優先されると人格指示が切り捨てられる可能性

**予測**:
- 軽いタスク: 両者とも人格反映度が高い
- 重いタスク: Claudeは人格を維持、Codexは人格が消失して「素のGPT」的な応答になる

---

## H4: ユーザー属性分岐仮説（定性的）

**主張**: Claude Code を好むユーザーとCodexを好むユーザーは、求めている体験が根本的に異なる。

| 属性 | Claude Code 好み | Codex 好み |
|------|-----------------|------------|
| 重視するもの | 対話・文脈・過程 | 結果・速度・自律性 |
| プロンプトスタイル | 自然言語、曖昧さ許容 | 構造化、明示的指示 |
| エージェントに求める役割 | 共同作業者 | 自動実行ツール |
| コンテキスト設計 | 分散型（複数ファイル） | 集約型（単一ファイル） |

**検証方法**: X/Twitter上のユーザー発言を収集し、定性分析を行う（実験4として別途実施）。
